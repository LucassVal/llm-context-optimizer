# 🧠 Lobo: Machine Memory

## 📍 Propósito
Centro de transformação de dados, criptografia e compressão semântica do NeoCortex. Todas as operações que convertem dados entre formatos humanos e de máquina são governadas por este Lobo.

## 🏷️ Semantic Scope (LEXICO)
| Atom | Meaning |
| :--- | :--- |
| `#ENC` | Operação de encriptação |
| `#DEC` | Operação de desencriptação |
| `#CMP` | Operação de compressão (LEXICO) |
| `#CNV` | Operação de conversão de formato |

## 🔒 Políticas de Acesso
- Apenas T0 e Guardian podem invocar operações `#DEC` e `#ENC`.
- Courier e Engineer podem invocar `#CMP` e `#CNV` apenas para formatos não sensíveis.
- Todas as operações `#DEC` são registradas no WAL com `severity: HIGH`.

## 🧩 Ferramentas MCP Associadas
- `neocortex_machine_memory` (a ser criada)
  - Ações: `convert`, `encrypt`, `decrypt`, `compress`, `decompress`, `get_dictionary`
💡 4. Exemplo de Uso na Prática
Cenário: O T0 precisa acessar a chave da API DeepSeek para fazer uma chamada.

Sem o Lobo Machine Memory:

O T0 lê a chave de algum lugar (variável de ambiente, arquivo de config).

O T0 usa a chave diretamente.

Risco: A chave pode vazar em logs ou ser acessada por um agente não autorizado.

Com o Lobo Machine Memory:

O T0 solicita ao Lobo: neocortex_machine_memory.decrypt(key_alias="deepseek_api_key").

O STEP 0 valida que o T0 tem permissão para essa ação.

O Lobo usa o CryptoHub para desencriptar a chave, que estava armazenada de forma segura (ex: no Firestore ou em um cofre local).

O Lobo retorna a chave apenas em memória para o T0.

O Lobo registra a operação no WAL.

Resultado: A chave nunca é exposta em texto puro em arquivos ou logs. O acesso é auditado.
