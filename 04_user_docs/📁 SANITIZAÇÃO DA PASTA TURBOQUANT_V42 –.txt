🔧 MCP (Model Context Protocol) – Detalhes técnicos
Servidor MCP principal
- Arquivo: neocortex_framework/neocortex/mcp/server.py
- Ferramentas: 22 ferramentas, 73+ ações
- Transportes: WebSocket (ws://localhost:8765) e stdio
- Alias no AGENTS.md: $mcp (atualmente aponta para neocortex_framework/DIR-MCP-FR-001-mcp-server/NC-MCP-FR-001-mcp-server.py, mas o arquivo real está em neocortex/mcp/server.py)
Diretório de ferramentas MCP
- neocortex_framework/neocortex/mcp/tools/ – Módulos individuais das ferramentas
- neocortex_framework/DIR-MCP-FR-001-mcp-server/ – Scripts auxiliares e testes
Configurações do MCP
1. Configuração para Antigravity
- Arquivo: antigravity_neocortex_config.json (raiz do projeto)
- Servidores definidos:
  - neocortex (stdio): inicia automaticamente via python -m neocortex.mcp.server --transport stdio
  - neocortex-websocket (WebSocket): conecta a ws://localhost:8765
- Ferramentas autoaprovadas: neocortex_cortex, neocortex_ledger, neocortex_report, neocortex_subserver, neocortex_task, neocortex_checkpoint, neocortex_config, neocortex_lobes, neocortex_search, neocortex_agent, neocortex_pulse
2. Configuração do sistema NeoCortex
- Arquivo: neocortex_framework/DIR-CFG-FR-001-config-main/neocortex_config.yaml
- Seções principais:
  - llm: provedor primário (Ollama), fallback chain (Ollama → DeepSeek → OpenAI), backends por agente (courier, engineer)
  - paths: mapeamento de aliases para diretórios DIR-*
  - cache: tamanhos de cache para ledger, manifest, hot cache
  - search: motor de busca (Xapian/SQLite FTS5)
  - scheduler: intervalos de pruning, consolidação, AKL, backup, checkpoint
  - monitoring: métricas ativadas, store DuckDB
Scripts de inicialização
- start_neocortex_mcp.bat (Windows) – inicia servidor WebSocket
- start_neocortex_mcp.ps1 (PowerShell) – mesma funcionalidade com logs
